精通CSS_阅读总结

发表于 2016-04-22

《精通CSS（第2版）》

今天把精通CSS书阅读完了，总的来说，这是一本还不错的书，对于HTML和CSS入门还是挺不错的。看网上很多人说翻译的不好，整个看下来很多术语确实和平时使用的不怎么一样，可能由于书也比较老了，但是不影响阅读。

阅读笔记

整个书的框架待整理。

Point

书的最后介绍了一些实例，用的技术有几个比较陈旧，改版之后的版本有增加CSS3的支持，这个还是不错的。
虽然用的标准不是CSS3或者HTML5，但是在书中都有提及到这方面的知识，改版后的也是与时俱进了。
看完书自己学习到了几个写CSS中需要注意的点，整理下：

构建CSS布局的流程

要想创建可伸缩且容易维护的CSS系统，首先应该检查设计，寻找重复的模式，这包括页面结构中元素反复出现的方式。

大致流程如下：

  1. 先把页面划分为大的结构性区域，比如容器、页眉、内容区域和页脚
  2. 内容区域本身，开始建立网格结构
  3. 在各个内容区域中寻找不同的布局结构
  4. 关注不同类型的内容
  5. 查看每个内容块的结构，看看不同的类型中是否有共同的模式
  6. 找出模式并确定命名约定后，马上开始定义将使用的元素

让设计居中的方法

  • margin: 0 auto (水平居中)
  • 设置line-height （垂直居中）
  • IE5.x和6中不支持margin：auto，但是它将text-align:center认为让所有东西居中，不只是文本。

宽屏窄屏布局

  • 为适应宽屏窄屏，通常将宽度设置为主题的百分之几，或者使用em相对于文本字号设置宽度
  • 固定宽度的布局
    • 常采用浮动
    • 不适应宽窄屏

流式布局

  • 尺寸用百分数设置
  • 添加min-width和max-width设置自适应的大小限制

弹性布局

  • 相对于字号来设置元素的宽度，以em为单位设置宽度，可以确保在字号增加时，整个布局随之扩大。
  • 大多数浏览器上的默认字号是16像素，10像素大约是16像素的62.5%，所以在主题上将字号设置为62.5%就可以，这样让1em相当于10像素
  • 内部宽度仍然使用百分数，只以em为单位设置容器的宽度

图像处理

  • 对于需要跨越大区域的图像，比如站点页眉或品牌区域中的图像，可以考虑使用背景图像而不是图像元素。
  • 对于常规内容的图像，希望他们垂直和水平伸缩以避免被剪切。可以将图像元素添加到没有设置任何尺寸的页面上。然后设置图像的百分数宽度，并且添加与图像宽度相同的max-width以避免图像失真。

选择器优先级（特殊性）

  • 不要随便添加更特殊的选择器，因为这可能会给代码的其他部分带来特殊性问题。更好的方法往往是删除额外的选择器，让他们尽可能一般化，只在需要细粒度的控制时添加更特殊的选择器。

bug查看方法

  • 隔离问题：在相关的元素上应用边框或轮廓，看看他们的反应。

CSS之清除浮动

发表于 2016-04-19

导读

学习CSS有一小段时间了，在练习的时候经常碰到布局问题，特别是和浮动有关的。才考虑用浮动做布局时，常常碰到一些”不受控制”的问题，让布局的效果往往不在意料之内，这个时候需要清除浮动来解决问题。那么，什么是浮动呢？我们为什么需要清除浮动？怎么清除浮动？这里，试着把学习到的总结一下，让自己对浮动有一个较为系统话的理解。

什么是浮动？

浮动（float）是一种可视化格式，在CSS中经常被用到，通过它我们可以很方便地布局。和布局相关的比较常用的应用有两列布局、两列自适应布局、文档环绕等。
在w3school中对浮动的描述如下：

浮动的框可以向左或向右移动，直到它的外边缘碰到包含框或另一个浮动框的边框为止。
由于浮动框不在文档的普通流中，所以文档的普通流中的块框表现得就像浮动框不存在一样。

为什么要清除浮动？

浮动会让元素脱离文档流，不再影响不浮动的元素。实际上，它不是完全的脱离文档流，而是半脱离文档流（对元素，脱离文档流；对内容，在文档流），什么意思呢？见下图：

图中对于文字内容，并没有脱离文档流，float仅仅是对元素有影响。也就是说，如果浮动的元素后面有一个文档流中的元素，那么这个元素的框会表现的像浮动根本不存在一样，但是框内的文本会受浮动元素的影响，会移动以流出空间。用技术术语来说，浮动元素旁边的行框被缩短，从而给浮动元素留出空间，因此行框围绕浮动框。这种方法通常适用于文本环绕图像。

这种运用可以很好的得到文本环绕的效果，但是当浮动元素的高度大于旁边一个框的高度，而影响到另外一个框时，这往往不是我们要的效果，如下图：

使用浮动可能导致的问题有：

  1. 背景不能显示
  2. 边框不能撑开
  3. margin 设置值不能正确显示

我们需要阻止后续的行框围绕在浮动框的外边，这时候就需要清除浮动了。清除浮动可以对包含这些行框的元素应用clear属性。
clear属性的值可以是left、right、both、none，它表示框的那些边不应该挨着浮动框。使用clear元素时，浏览器会在该元素顶上添加足够的外边框，使元素的顶边缘垂直下降到浮动框的下面。

clear语法：

clear : none | left | right | both

取值：

none  :  默认值。允许两边都可以有浮动对象

left   :  不允许左边有浮动对象

right  :  不允许右边有浮动对象

both  :  不允许有浮动对象'

需要注意的是，clear属性仅仅会影响使用元素的本身，而不会对其他元素造成影响。
如下图所示：

总而言之，浮动元素脱离了文本流，不影响周围元素，清除浮动实际上为前面的浮动元素留出了垂直空间。

怎么清除浮动？

方法一：

在最后一个段落下面添加一个空元素并清理它，这样可以迫使容器元素包围浮动元素。

.news {
  background-color: gray;
  border: solid 1px black;
}

.news img {
  float: left;
}

.news p {
  float: right;
}

.clear {
  clear: both;
}

<div class="news">
    <img src="news-pic.jpg" />
    <p>some text</p>
    <div class="clear"></div>
</div>

优点：
简单易用，容易理解
缺点：
添加不必要的代码，需要添加无意义的标记，违背了我们结构和表现分离的原则，不建议使用。
ps:
方法一通常除了添加一个空div外，有的时候会添加
标签，使用br标签本身的clear属性来清除浮动，这样相对结构性好一点，但是结构和表现还是不分离的，同样不建议使用。

方法二

不对浮动的文本和图像进行清理，选择浮动div容器

.news {
  background-color: gray;
  border: solid 1px black;
  float: left;
}

.news img {
  float: left;
}

.news p {
  float: right;
}

<div class="news">
    <img src="news-pic.jpg" />
    <p>some text</p>
</div>

优点：
实现效果的代码简单。
缺点：
下一个元素会受到这个浮动元素的影响。
ps:
有人会通过浮动布局中几乎所有的东西来解决这个问题，但是这样浮动就会变得很复杂。

方法三

对父元素设置overflow属性。

overflow属性定义在包含的内容对于制定的尺寸太大的情况下元素应该怎么样。

overflow值为hidden和auto（注意: 仅这两个值）的overflow属性有一个很有用的副作用，它会自动地清理包含的任何浮动元素。

优点：不存在结构和表现分离的问题。
缺点：设置了overflow属性值会影响它本来的表现，也就是产生overflow本身的效果，例如产生滚动条或截断内容等。
ps:
在IE6等低版本浏览器中，还需要注意兼容性，出发hasLayout才能使用，例如zoom:1。

方法四（较常用）

使用CSS生成的内容或者JavaScript对浮动元素进行清理，并不直接向标记中添加标签元素进行清理，而是将它动态地添加到页面中。
例如指定进行清理的元素出现在哪里，常常需要添加一个类名，然后对该类名设置clearfix属性清理。

<div class="news clearfix">
    <img src="news-pic.jpg" />
    <p>some text</p>
</div>

结合使用:after伪元素选择器和内容声明在指定的现有内容的末尾添加一个新的内容。常常为一个点（这样不引人注意），并将它设置为块级元素（display:block 使生成的元素以块级元素显示,占满剩余空间，这样才能改变它的高度），然后将它的height设置为0（避免生成内容破坏原有布局的高度），visibility设置为hidden（使生成的内容不可见，并允许可能被生成内容盖住的内容可以进行点击和交互;）,overflow设置为hidden,使内容不被溢出。因为被清理的元素在他们的顶外边距增加了空间，所以生成的内容需要将它的display属性设置为block。

.clearfix:after { 
    content: ".";
    display: block;
    height: 0;
    clear: both;
    visibility: hidden; 
    overflow: hidden;
}

优点：
不需要增加额外的元素。结合和表现分离
缺点：
需要添加类名。IE6-7不支持:after，使用 zoom:1触发 hasLayout。

方法五

父元素设置display:table

优点：
结构语义化完全正确，代码量极少。
缺点：
盒模型属性已经改变，由此造成的一系列问题，得不偿失，不推荐使用。

补充

伪元素:before和:after在清除时的区别：

:before通常用于清除Margin Collpase。
那什么是Margin Collpase呢？其实就是我们俗称的外边距叠加。简单地说，当两个或更多垂直外边距相遇时，他们将形成一个外边距，这个外边距的高度等于两个发生叠加的外边距的高度重的较大者。

它具备的两个特点是：

  1. 普通文档流中的块级元素才会发生叠加，行内元素、浮动元素和绝对定位元素不会发生叠加。
  2. 垂直外边距上的叠加。

我们常常会使用before来清除叠加行为，在某两个边框中间添加一个不会显示的元素来清除叠加，使得外边距距离不缩短。

原理上阐述产生浮动的原因

float等元素会触发BFC（Block formatting contexts （块级格式化上下文））。
BFC从通俗上说就是一个单独的盒子，里面的子元素不会影响外面元素的布局，同时还存在于普通文本流中。
关于BFC如何触发及其特性的讲解可以看这篇文章————那些年我们一起清除过的浮动

windows下puer安装

发表于 2016-04-17

puer简介

Puer是一个可以实时编辑刷新的前端服务器。具体可以看puer

windows下puer安装方法简介

安装node.js

首先，去node.js官网下载node.js并安装，根据你自己的机型下载对应版本，然后一直点下一步默认安装。
其中，node.js具有两个版本，LTS和Stable，其中LTS版本比较稳定安全，Stable版本迭代更新比较快。具体可以看下这篇文章中英文解释（ps，中文翻译一般）————nodejs v4.4.2 LTS和 v5.10.1 Stable 有什么区别？

安装puer

node.js安装好后，打开Node.js command prompt，使用全局命令安装puer：

npm install puer -g

执行完后，puer就安装好了，具体的使用方法可以输入puer-h查看help。

使用puer

打开Node.js command prompt或者命令行cmd或者terminal等，进入到当前需要实时编辑刷新的文件所在的文件夹，然后输入命令puer就可以使用puer进行实时编辑了。

cd /path/workspace
puer

puer会默认打开http://localhost:8000/页面，并贴心的帮你列出了所有你本地可用的ip以及当前的地址二维码，方便手机访问。

以上就是前端新人安装puer的过程了~
具体的细节和进阶使用请看超简单工具puer——“低碳”的前后端分离开发
