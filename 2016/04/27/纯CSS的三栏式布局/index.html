纯CSS的三栏式布局

发表于 2016-04-27   |   分类于 TO前端   |     |   阅读次数

三栏式布局

最近在做百度前端技术学院的题，发现自己对CSS熟悉度还不够，还有很多的坑没有踩。
三栏式布局是百度前端技术学院的Task2，在此就以这个为引出，基本实现根据此任务，来说说三栏式布局的实现方法。

实现方法

方法一：float

方法思想

使用浮动可以让元素脱离文档流，实现布局和定位。
实现的思想很简单，就是左边元素左浮动，右边元素右浮动，中间使用margin，设置marginleft和right来实现自适应。

踩过的坑

三栏的顺序

三个元素的顺序的关键是要把中间div放在最后，左右栏div顺序随意。
为什么呢？
这个就看我们对float的理解了。
元素在浮动时，是相对浮动的，它虽然会脱离它当前所在的文档流，但是会根据它前面元素的位置来决定浮动的位置。也就是说，它的浮动的上面位置是和它前一个原来在标准流中的元素底部对应，而不是包裹它的外面的元素。
这部分具体的理解可以看css浮动

假如某个div元素A是浮动的，如果A元素上一个元素也是浮动的，那么A元素会跟随在上一个元素的后边(如果一行放不下这两个元素，那么A元素会被挤到下一行)；如果A元素上一个元素是标准流中的元素，那么A的相对垂直位置不会改变，也就是说A的顶部总是和上一个元素的底部对齐。

回到我们当前的demo，如果我们把中间的div放置在了左右两栏的中间，那么最后一个元素在浮动的时候，会浮动在中间div的下面（不管它是左浮动还是右浮动），因为最后一个元素它的浮动的位置是它在标准流中间前一个元素（即中间div）的底部。这样永远也达不到我们要的效果。
ps：表示自己最初对float理解不透彻，把自己坑惨了。

清除浮动

这个坑大家都应该了解的比较多了，唯一需要注意的就是对于CSS的清除浮动(clear)，一定要牢记：这个规则只能影响使用清除的元素本身，不能影响其他元素。
所以我们通常采用给外层div加上一个after选择器，增加一个元素让它清除浮动来使得外层div撑开。
通过这样的方式能满足：改变中间一栏的内容长度，以确保在中间一栏较高和右边一栏较高时，父元素的高度始终为子元素中最高的高度。
噢，不要忘记了，清除浮动还需要照顾一下浏览器的兼容性，使用zoom触发bfc就好啦！

方法二：position

方法思想

使用绝对定位和相对定位来构建三栏式布局在横向上可以很好的满足要求。思想就是通过给外层的div设置相对定位，然后内层的左和右边的两栏设置绝对定位（相对于外层div的绝对定位），然后中间用margin设定位置，这样可以实现三栏布局。

实现效果见demo
注意这里三栏元素先后顺序不影响布局。

踩过的坑

高度适应问题

使用position最大的问题就是高度上的适应，使用absolute属性会导致高度上的坍塌，也就是说，给左右两栏设置absolute后，外层div的高度和这两个绝对定位的元素已经没关系了，如果要实现高度适应的效果，提供一个min-height是最好的解决方法。
关于相对、绝对定位存在的问题和详细解释见CSS 相对|绝对(relative/absolute)定位系列
一般我们不使用position来对页面内部的元素进行具体的布局，这样容易导致问题且扩展性低。position一般适合做整个页面上固定位置的东西，尽管有相对定位下的绝对定位，但是也不要忘记它导致的高度坍塌问题。

方法三：margin

方法思想

margin方法在实际中用的还比较多。这种方法通过把所有元素浮动，然后通过改变浮动的位置来实现。需要注意的就是要把中间的内容区域放在最前面，使用双层标签，外层div宽度100%显示，并且浮动浮动（给的demo为左浮动，下面所述依次为基础），内层div为真正的主体内容，然后定义其margin值，外层的边框隐藏，为none，给内层设定边框。左栏与右栏都是采用margin负值定位的，左栏左浮动，margin-left为-100%，由于前面的div宽度100%与浏览器，所以这里的-100%margin值正好使左栏div定位到了页面的左侧；右侧栏也是左浮动，其margin-left也是负值，大小为其本身的宽度。

demo

实现效果见demo
注意三栏元素的顺序，中间在前即可。

踩过的坑

顺序问题

最大的坑就是顺序问题啦，这个需要好好的把这个方法的思想理解。然后过多的margin定位导致的位置也不一样，还需要注意负margin的时候要考虑border的宽度对它的影响。
这种方法比较实用，但是就是理解比较困难，上手难一点，然后出问题了不好找bug啊~！

自适应宽度问题

还有一个坑就是宽度自适应的问题，当把浏览器调小，设定元素的min-width的时候一定要注意是中间的min-width加上左右固定好后的width，不然就出现右侧的和中间的内容叠加啦！

方法四——flex

中心思想

使用弹性布局的方法来实现，通过指定中间内容分配全部的剩余空间来获得自适应的宽度。通过flex-basis就能满足我们需要的min-width功能啦。那自适应高度怎么弄呢？解决margin的方法，给所有的元素增加双层标签，外标签高度不管，边框隐藏，然后给内标签设置定位就好了，外标签的主要作用就是用来分配横向的剩余空间的。因为flex分配剩余空间后，如果不指定height值，它会默认是外层div的高度，也就是最大的那个高度，这个肯定不是我们想要的。设置双层标签可以满足内层边框高度是自适应的。

demo

实现效果见demo。

踩过的坑

高度自适应问题

最大的坑就是高度的适应问题。一般我们需要元素的边框高度就是他自己的内容高度，但是我们发现用flex后，它的高度变成了整个div的高度。通过双层标签，把外面的高度隐藏，内层边框的就是自适应的啦~！

总结

纯CSS实现三栏式布局的方法有这么多（可能还有遗漏的），一般我们比较常用的就是float和margin两个方法，float保证浮动的清除就可以很好的解决问题，margin呢就是比较难以理解，理解了以后也是很好用的。
学习最快的接收方式就是实践，多试试就知道解决办法了~！
不过关于各种方法的优缺点上理解还需要更为深入，等学到后面再多多了解。
