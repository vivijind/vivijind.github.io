JavaScript对象

发表于 2016-04-26   |   分类于 TO前端   |     |   阅读次数

对象是个什么东西？

对象，通俗一点理解其实就是我们生活中的事物，比如人、房子、车子等。每一种事物都有自己的属性。
在面向对象的语言中，通常通过类来创建任意多个具有相同属性和方法的对象。
ECMA-262把对象定义为：“无序属性的集合，其属性可以包含基本值、对象、函数”。

JS中并没有类的概念，它的对象和其他拥有类的语言创建方法不太一样。那js中的对象应该怎么创建呢？下文主要讲ECMAScript5中的对象怎么创建，也是把自己最近学的东西整理整理，好记性不如烂笔头嘛，写出来往往是理解东西最快的方式。

PS:由于ECMAScript6中为了弥补js没有类这一概念的不足，增加了class类型，本文在此不做讨论。不过，稍微了解了下ECMAScript6，有了class这个概念，对象也越来越容易被理解和使用了，希望这个会被推广使用。class和extends，类和继承，这两种思想也使得js和其他语言越来越相近。

最简单的创建方法

早期，大家都用new一个object和对象字面量的方法来创建对象，这是最简单的两种创建方法。object就是js自带的对象，new一个object就可以创建一个自定义的对象了，然后可以对该对象增加属性和方法。对象字面量把属性和方法整合到一起，代码相对简洁，实际上js在执行时也是先创建一个object，然后给其赋值。

缺点
虽然object可以很方便的创建对象，但是每次创建都需要重复的赋值属性，重复的给对象提供方法，这样的方式会产生大量的重复代码。
为了防止重复代码的出现，人们开始借鉴工厂模式来创建js的对象。

工厂模式

工厂模式是一种很普遍的设计模式，理解也很简单，可以比喻为现实中的工厂，里面有很多车间（方法）用于创建各种事物，例如车，当我们需要产出一辆宝马，这时候我就通知工厂，在某个车间执行这个命令，也就是调用工厂方法，来产出宝马（创建一个对象），这个方法的实际就是把创建对象的方式封装起来，提供一些属性命令，返回一个对象，这样就得到我们需要的对象了。
如果创建对象的过程比较复杂，那将这部分代码封装起来很有用，也能防止代码的重复。

缺点
工厂方法虽然解决了创建对象代码重复的问题，但是不能通过该方法获得对象的具体类型。

借用构造函数

构造函数这个概念来自于其他语言。一般我们借用构造函数来构建对象都是指自定义的构造函数。
一般，我们以一个大写字母开头来表示构造函数，用于区分（借鉴自其他OO语言）。
构造函数特点：

  • 没有显式的创建对象
  • 直接将属性和方法赋给了this对象
  • 没有return语句
    构造函数调用过程：
  • 创建一个对象
  • 将构造函数的作用域赋给新的对象（this指向这个新对象）
  • 执行构造函数中的代码（为新对象添加属性和方法）
  • 返回新对象

创建原型对象

通过构造函数的方法创建对象这样一看问题也很多啊，那再怎么办呢？我们可以通过创建原型对象来解决问题。
每一个函数都有一个原型属性，也就是prototype，这个属性是一个指针，它指向一个对象，这个对象包含所有能被实例共享的属性和方法。
也就是说，其实所有的函数里面都有一个指针，它指向一块单独的内存，这块内存就是原型对象，这个指针就是prototype。通过这样一种方式，我们就能让所有的实例共享这一个内存。
基本原理：

  • 读取某个对象的某个属性时执行一次搜索，目标是具有给定名字的属性。
  • 从对象实例本身开始，找到，返回本身的值。
  • 没找到，搜索该指针指向的原型对象，找到，返回原型对象中的值。

注意：可以通过对象实例访问保存在原型中的值，但不能通过对象实例重写原型中的值。

简单一点儿
每添加一个属性和方法都要写一遍XX.prototype，好烦啊~！So，我们可以用一个包含所有属性和方法的对象字面量来重写整个原型对象。

注意：重写原型对象切断了现有原型与任何之前已经存在的对象实例之间的联系，之前的那些仍然引用的是最初的原型。

缺点
原型对象中，属性都是共享的，那么对于引用类型的数据，共享的则是它的引用（指针），如果我们操作当前的引用类型，实际上修改了该引用类型所在的内存里的内容，不能保证每个对象有自己共享的引用类型内存。

修改实例对象的属性和操作实例对象的属性还是有点不一样
landRover.abc=xxx 这种是修改对象的属性，landRover.abc.push(xxx)则是操作对象的属性
无论原型上的abc是什么属性，前者的修改都不会影响到原型上的abc属性
而后者是直接操作的原型上的abc属性，实例对象上并没有abc属性

构造函数+原型模式（常用方法）

既然引用类型放原型模式上会导致操作修改的问题，那么放构造函数里就不会啦，因为构造函数的问题也就是每个东西有自己的副本。

构造函数模式用于定义实例属性，而原型模式用于定义方法和共享的属性。结果，每个实例都会有自己的一份实例属性的副本，但同时又共享着对方法的引用，最大限度的节省了内存。

另一种常用方法：动态原型模式

把所有信息都封装在了构造函数中，通过在构造函数中初始化原型，又保持了同时使用构造函数和原型的优点。
可以通过检查某个应该存在的方法是否有效，来决定是否需要初始化原型。

注意：在使用动态原型模式时，不能使用对象字面量重写原型。

稳妥构造函数模式

没有公共属性，而且方法也不引用this对象。
适用：在一些安全的环境中（比如禁用this、new），或者防止数据被其他应用程序改动时适用。

总结

一般常用的方法为构造函数与原型混合的模式，这是目前在ECMAScript中适用最广泛，认同度最高的一种创建自定义类型的方法。可以说，这是用来引用类型的一种默认模式。
