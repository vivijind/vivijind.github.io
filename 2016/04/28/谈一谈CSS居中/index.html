CSS水平垂直居中

在很多时候我们都会用到水平居中、垂直居中的布局，例如最常见的登录框。那有什么解决办法呢？有很多资料解释的比较清除，这里自己总结一下。

水平居中

内联元素（inline)

首先，如果这个元素是内联元素，就像文本、链接等这些，那么居中的方法就用：

常见的内联标签有：span a img input label br 等

定宽+块级元素（block）

如果这个元素是块级元素且定宽，就像文本、链接等这些，那么居中的方法就用：

常见的块级元素：div h1 ~ h6 table p ul li 等

不定宽+块级元素（block）

如果元素不知道宽度，且为块级元素，水平居中方法有三种。

方法一：使用table标签

  1. 为需要设置的居中的元素外面加一个table标签
  2. 为这个table设置“左右margin居中”（类似定宽+块级）

方法二：display：inline

改变块级元素类型，将其设置为inline类型，然后使用text-align: center来实现居中效果。
这种方法优势在于不用增加无语义标签，但是变成内联元素不能设置长度值，有内联元素的缺陷。

方法三：float+position

  1. 给父元素设置float，并设置positon：relative，且left：50%
  2. 子元素设置position：relative和left：-50%(或者是margin-left设置为-50%)
    这种方式保留块级元素的特性，但是使用相对定位也存在一定问题。

多个块级元素

如果元素为多个块级元素，然后想排在一行，且水平居中，有两种方法。

方法一：inline-block方法

  1. 父元素设置text-align:center
  2. 设置所有子元素display为inline-block,然后变成内联元素后设置text-align: center来实现居中效果。

方法二：flex

设置display:flex，使用弹性布局，并设置对齐方式为居中对齐justify-content: center;

垂直居中

高度确定+单行文本

父元素高度确定的单行文本的竖直居中的方法是通过设置父元素的 height 和 line-height 高度一致来实现。

高度确定+多行文本、图片、块级元素

父元素高度确定的多行文本、图片、块级元素的垂直居中的方法有两种。

方法一：table+vertical-align:middle

使用适用于表格的vertical-align方法来使得元素垂直居中生效。
或者说可以设置display为table-cell，激活vertical-align属性。（ie67不兼容）

方法二：改变display类型

设置positon：absolute或者float:left/right，将元素自动变为inline-block方式显示。

实际练习踩到的坑

发现自己水平和垂直居中还存在问题也是在所百度前端技术学院的题（task4）的时候发现的，我可以很好用一种方式实现效果，但是多种方式的时候，发现自己还存在问题，或者说有的理解有问题。在这里mark一下。

实现注意事项：

  • 思考不同情况下（如灰色高度是根据内容动态变化的）水平垂直居中的解决方案。
  • 动手试一试各种情况的组合，父元素和子元素分别取不同的 position 值。思考 position 属性各种取值的真正含义，尤其是 absolute 究竟是相对谁而言的。
  • 注意测试不同情况，尤其是极端情况下的效果。
  • 调节浏览器宽度，灰色元素始终水平居中。
  • 调节浏览器高度，灰色元素始终垂直居中。
  • 调节浏览器高度和宽度，黄色扇形的定位始终准确。

实现方法一

最简单的满足题目要求的方法就是使用positon为fixed（或者absolute\relative，效果一样），fixed表示相对于浏览器的定位，将灰色框设置left、top为50%就能很好的解决问题了。
不过要注意，position的定位是对框框的左上角和浏览器窗口左上角的定位，要保持中心居中还需要设置transform变形移动下或者margin负值移动一下。

关于html、body宽高设置为100%

div的100%是从其上一级div的宽高继承来的，有一点很关键，就是要设置div100%显示，必须设置其上一级div的宽度或高度，否则无效。
前面总得有个容器说明他的高度是多少。这样的话div才能按比例100%继承上一级的高度可惜的是浏览器一般默认解释为内容的高度，而不是100%。解决方法：只要为html和body设置高度为100%就可以了html,body{height:100%;}这样之后div会按比例来继承上一级的高度了.仅仅设置的DIV元素的height属性貌似没有什么效果。

总结一下：浏览器默认没有高度属性，一般默认为内容的高度而不是100%。但是我们往往需要以浏览器100%的宽度和高度来做参考，或者说div继承这个属性，然后在此做定位。所以我们往往给设置html和body的宽高为100%。同时，html不等于body，其中firefox中的html标签不是100%高度，所以我们需要给html和body标签都设置。

实现方法三

table方法，指定display为table-cell，这种方法历史很悠久了。需要注意html部分相对于方法三又加了一层父元素。最外层父元素设置display为table，然后内层指定display为table-cell，让标签元素以表格单元格的形式呈现，然后最内层指定为相对定位给corner元素提供定位的参照。
