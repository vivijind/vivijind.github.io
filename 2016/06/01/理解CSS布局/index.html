理解CSS布局

发表于 2016-06-01   |   分类于 TO前端   |     |   阅读次数

CSS布局

CSS真的是个很强大的东西，它可以让没有样式的文字、图形等具有他们自己的样子。如果没有CSS，想想我们的网页会发生什么呢？简直是disaster。
CSS布局也就是让页面上所有的东西服服帖帖，按照我们想要的给他们安排位置。
常见的CSS布局有：水平垂直居中、多列、全屏、响应式等等。这么多布局，我们是不是要一个一个记下来呢？其实万变不离其中，需要搞清楚的是在布局中起作用的css属性的每个特性，然后按照特性来对布局进行分配就能解决一大部分情况了。
在对CSS进行布局时，首先将问题分解，然后考虑每一个特性怎么实现，最后结合整理形成最终的布局。
下面我们对布局中一些常见的问题来进行具体的分析。

先说一些基本布局元素及其特性

display

块级元素 display:block

常用的块级元素有: div、p、form、header、footer、section、h1~h6、ul等
块级元素的特性：

  1. 默认宽度为父元素的宽度（影响元素大小）
  2. 可以设置宽高（影响元素大小）
  3. 换行显示（影响元素位置）

    行内元素display: inline

    常用的行内元素有：span、a、label、cite、em等
    行内元素的特性：
  4. 默认宽高为内容宽高（影响元素大小）
  5. 不可设置宽高（影响元素大小）
  6. 同行显示，可以在行内换行（影响元素位置）

    结合块级和行内元素 display:inline-block

    常用的inline-block元素有：input、textarea、select、button等
    inline-block元素特性：
  7. 默认宽高为内容宽高（影响元素大小）
  8. 可以设置宽高（影响元素大小）
  9. 同行显示（影响元素位置）
  10. 整块换行（范围超过边界时，会整个一起换行，不会在行内换行）（影响元素位置）

    display:table

    指定对象作为块元素级的表格,类同于html标签table

    display: table-cell

    指定对象作为表格单元格。类同于html标签td

    display: none vs visibility:hidden

    display: none不删除元素的情况下隐藏或显示元素,不会保留元素本该显示的空间
    visibility:hidden: 保留元素本来该显示的空间

position定位

优点：绝对定位脱离文档流，子元素不会对其他元素产生影响
缺点：

  • 需要设置宽度，并进行具体的left设定来水平排列
  • 兼容性问题，有些浏览器需要加私有前缀

static

static 是默认值。任意 position: static; 的元素不会被特殊的定位。一个 static 元素表示它不会被“positioned”，一个 position 属性被设置为其他值的元素表示它会被“positioned”。

relative

  • 元素仍在文档流中
  • 参照物为元素本身，原来的位置还在，别的元素就不能替代这个位置
  • 常用场景：改变z轴上的层级、绝对定位元素的参照物

absolute

  • 默认宽度为内容宽度
  • 脱离文档流
  • 参照物为第一个定位祖先（祖先上需要有定位元素）/根元素

fixed

  • 默认宽度为内容宽度
  • 脱离文档流
  • 参照物为视窗

float浮动

  • 默认宽度为内容宽度
  • 脱离文档流（和position不同，会被边界挡住，不是完全脱离）
  • 向指定的方向一直移动
  • 问题：需要清除浮动

flex

弹性布局，设置或检索弹性盒模型对象的子元素如何分配空间。
这个属性很好用，就是存在兼容性问题。

定宽高和宽高自适应

定宽也就是给定元素的宽度
宽度自适应大概是我们通常所说的根据内部元素内容来适配宽度（可以说宽度未知），或者说是根据屏幕、外层元素宽度来适配宽度。
概念搞清楚了，先说说怎么让一个元素满足这两个条件。

定宽高

定宽高===给定宽高===display：block
在CSS中，只有块级元素才能给定宽高。
所以，为了给元素定宽高，需要将元素设置为块级元素。

宽高自适应（根据内容）

设置为行内元素，可以的方式有:inline、inline-block

宽高自适应（根据父元素）

具有块级元素的特性（首先可以设置宽高），然后宽高设置为百分比即可

水平垂直对齐的方法（居中等）

水平布局

margin

在使用margin之前，要明白margin：auto的计算原理，它是通过元素本身的宽度和外层元素的总宽度计算得到，属性为auto时，浏览器会根据宽度自动计算，然后分配，所以可以居中。
那么，在使用时，也就需要给定元素的宽度才能让margin：0 auto生效，浏览器需要先知道元素宽度，还需要知道元素所对应居中的区域的外层宽度。
margin：auto使用条件：

  1. 块级元素
  2. 在子元素上使用，根据已有宽度相对于父元素的居中

text-align

使用条件：

  1. 设置文本或img标签等一些内联对象（或与之类似的元素）的对齐方式
  2. 在需要设置的对象父元素上设置
  3. 需要注意该属性为继承属性

flex

justify-content 设置对齐方式

绝对定位和相对定位

通过定位（四个方向）一点点分布对齐

垂直布局

vertical-align

使用条件：

  1. 设置文本或img标签等一些内联对象（或与之类似的元素）的对齐方式
  2. 在需要设置的对象父元素上设置
  3. 需要注意该属性为继承属性

flex

align-items设置对齐方式

绝对定位和相对定位

通过定位（四个方向）一点点分布对齐

margin

设置不同的margin top\bottom值，但是需要注意margin的叠加问题（需要知道宽高）

水平排列

实现有三种方式，一种是元素为行内元素，一种是定位布局，另一种是弹性布局。

display:inline inline-block

可以通过修改display属性值转行块元素和行内元素：
子元素直接指定为inline-block（不能为inline，因为高度需要为自适应，inline不能指定高度，不满足题目中高度未知的要求）

display: table-cell

利用表格布局，子元素为table的单元格，即table-cell，这样子元素就一行排列，但是，table布局只是一个宽度根据内容走的类block元素，也就是宽度可设置但是高度不可设置，不满足题目中高度未知的要求。

绝对定位和相对定位

绝对相对定位布局（脱离文档流的方式）：这种需要知道每一个子元素的宽度，通过计算每个子元素宽度的偏移量来对子元素进行水平排列。
父元素display设置相对定位，子元素设置绝对定位，然后把子元素的left或者margin-left根据宽度来逐渐偏移。

flex布局

将父元素display设置为flex，那么所有子元素均水平排列。

垂直排列

块级元素

设置display：block或者display:table，块级元素默认垂直排列

绝对定位和相对定位

一点点计算

flex

将父元素display设置为flex，且flex-direction:column那么所有子元素均垂直排列。

总结

在考虑CSS布局问题时，先将问题分解，了解每一种布局需要什么要求和特性，然后找对应的方法来进行整理分配，最后调整修改一些可能存在的bug

实例分析

一个水平排列的列表，每项高度都未知，但要求底部对齐

问题分解

水平排列、高度未知、底部对齐

分析

水平排列、高度未知，实现有三种方式，一种是元素为行内元素，一种是定位布局，另一种是弹性布局。

  1. 可以通过修改display属性值转行块元素和行内元素：
  • 子元素直接指定为inline-block（不能为inline，因为高度需要为自适应，inline不能指定高度，不满足题目中高度未知的要求）
  • 利用表格布局，子元素为table的单元格，即table-cell，这样子元素就一行排列，但是，table布局只是一个宽度根据内容走的类block元素，也就是宽度可设置但是高度不可设置，不满足题目中高度未知的要求。
  1. 绝对相对定位布局（脱离文档流的方式）：这种需要知道每一个子元素的宽度，通过计算每个子元素宽度的偏移量来对子元素进行水平排列。
  • 父元素display设置相对定位，子元素设置绝对定位，然后把子元素的left或者margin-left根据宽度来逐渐偏移。
  1. flex布局
    将父元素display设置为flex，那么所有子元素均水平排列。

底部对齐：其实是垂直居中的变相解法

  1. 针对行内元素，设置底部对齐的方法有：
    vertical-align，针对行内元素设置垂直对齐方式，设置为bottom
  2. 针对定位布局
    对所有子元素设置bottom：0即可，这种情况是相对父元素的bottom，就是和父元素底部对齐。
  3. flex布局
    使用flex特性，align-items设置为flex-end即可底部对齐

综上所述，主要的方法有三种：

  1. 子元素设置：display：inline-block + vertical-align:bottom
    优点：兼容性好
    缺点：child继承父元素的vertical-align
  2. 父元素：position:relative + 子元素：position:absolute + bottom：0
    优点：绝对定位脱离文档流，子元素不会对其他元素产生影响
    缺点：
    需要设置宽度，并进行具体的left设定来水平排列
    兼容性问题，有些浏览器需要加私有前缀
  3. 父元素：dispaly:flex + align-items:flex-end
    优点：只需要设置parent结点，child自动称为flex item
    缺点：兼容性问题，css3新属性

最后

简单的讲了下自己对布局的理解，本来想结合实际过程来演示的，后来发现自己的表达能力还是有限，很多东西感觉还是没有表达清楚，实际过程中使用还是有很多问题需要去解决，但是只要每个属性的特性掌握了，就可以完成基本的布局。
